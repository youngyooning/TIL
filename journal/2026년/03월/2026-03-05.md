## ⚠️ Trouble
- **문제** : 커밋 메시지를 잘못 작성했는데 수정 방법이 바로 떠오르지 않았다.
- **원인** : Git을 단순 명령어 암기를 통해서만 알고 있언던 "과거의" 나.
- **해결** : `git commit --amend -m "새로운 커밋 메시지"` 명령어로 수정했다.
- **인사이트** : push 전이라면 `git commit --amend` 명령어로 가장 최근 커밋의 커밋 메시지를 수정 가능하다. 이전 커밋에 덮어씌우는 방식이다. 이미 push한 경우 `git push --force` 명령어를 사용한다.
