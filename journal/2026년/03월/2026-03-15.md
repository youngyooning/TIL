## ⚠️ Trouble
### 문제
Tailwind CSS 초기 설정을 위해 `npm install -D @tailwindcss/vite` 명령어를 실행했는데 설치가 실패했다.\
npm이 dependency tree를 해석하는 과정에서 peer dependency 충돌을 감지하고 설치를 중단했다.

#### 에러 메시지
```
npm error code ERESOLVE
npm error ERESOLVE unable to resolve dependency tree
npm error Found: vite@8.0.0
npm error Could not resolve dependency:
npm error peer vite@"^5.2.0 || ^6 || ^7" from @tailwindcss/vite@4.2.1
```

### 원인
Vite와 `@tailwindcss/vite` 사이의 버전 호환성이 일치하지 않는 것이 원인이였다.\
`vite@8.0.0`을 사용하는 프로젝트인데 설치하려던 패키지는 아직 vite@8을 공식 지원하지 않는 상태였기 때문에 npm이 충돌로 판단한 것이다.

### 해결
vite@8을 vite@7로 내려서 `@tailwindcss/vite`가 지원하는 버전대로 맞췄다.
1. `package.json`의 `vite` 버전을 `^7.1.0`으로 수정
2. 기존 설치 흔적 정리 (`rm -rf node_modules package-lock.json`)
3. 재설치 (`npm install`)

`@tailwindcss/vite`의 peer dependency 범위 안에 들어오게 되어 정상적으로 설치됐다.

### 인사이트
프로젝트를 처음 세팅할 때 무조건 최신 버전을 사용하는 것보다 생태계 호환성이 검증된 조합을 사용하는 것이 중요하다는 것을 배웠다.
