## ⚠️ Trouble
- **문제** : 프로젝트 기능 명세를 작성하면서 `사용자 시나리오 - 전제 조건 - 메인 플로우 - 상태/데이터 변화 - API/데이터 계약 - 화면 - UI 변화 - 예외 - 테스트 시나리오`와 같은 상세한 구조를 적용하다 보니 명세 작성 시간이 너무 오래 걸렸다.
- **원인** : 기능 13개가 담긴 MVP 범위가 생각보다 컸다. 또한 완벽한 명세를 한 번에 완성하려는 욕심이 작성 속도를 늦추는 원인이 되었다.
- **해결** : MVP 범위를 한 번 더 쪼갰다. 지금 당장 구현할 기능과 나중에 구현할 기능을 분리했다. MVP를 3단계로 나누고 첫 배포 목표를 1단계까지만으로 잡았다. 테이블도 모두 다 설계하지 않고 최소 핵심 테이블만 작성 후 나중에 추가하는 방식으로 진행한다.
- **인사이트** : 처음부터 완벽한 명세를 만들기보다 우선 개발이 가능한 수준으로 정리하고 점진적으로 다듬는 방식이 더 효율적이라는 것을 배웠다.
