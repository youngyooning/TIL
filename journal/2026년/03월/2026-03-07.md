- **프로젝트 기술 스택**
	- TanStack Query와 Zustand를 사용하기로 결정했다. 서버 상태는 TanStack Query로, 클라이언트 전역 상태는 Zustand로 상태 관리 역할을 분리했다.
	- TanStack Query : 서버에서 가져온 데이터 (`members`, `boards`, `columns`, `cards` 등)
	- Zustand : UI 전역 상태 (모달 열림 여부, 현재 선택한 카드, 필터 UI 상태 등)
- **RLS 정책 정리**
	- 역할은 Owner, Member 2개로 구성했고 RLS를 위해 권한 규칙 초안을 작성했다.
	- 기본적으로 사용자는 멤버로 등록된 워크스페이스의 데이터만 읽고 쓰기 가능하다. 즉, `workspace_members`에 존재하는 사용자만 워크스페이스에 접근 가능하다.
